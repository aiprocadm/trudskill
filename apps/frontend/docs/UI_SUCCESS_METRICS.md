# Метрики успеха доработки UI (cdoprof)

Документ фиксирует, **что измерять** после внедрения плана по интерфейсу. Сбор данных — на стороне продукта (аналитика, опросы, поддержка).

## Сценарии и время

- **Администратор / методист**: время от входа до открытия нужного раздела (курсы, группы, пользователи) — цель: сокращение за счёт крошек и навигации.
- **Слушатель**: время до списка «Мои курсы» и до карточки курса с понятным следующим шагом.
- **Ошибки загрузки**: доля сессий с повторным запросом после тоста «Не удалось загрузить данные» (косвенно — успешность сети и API).

## Качество навигации

- Обращения в поддержку вида «не нашёл в меню» / «где раздел X».
- Использование хлебных крошек (клики по родительским уровням), если включена веб-аналитика.

## Стабильность визуала

- **Рекомендация**: добавить 3–5 визуальных снапшотов (Playwright `toHaveScreenshot`) для эталонных страниц: главная дашборд, список пользователей, «Мои курсы», настройки, форма входа.
- Запуск в CI при появлении инфраструктуры артефактов и эталонных изображений.

## Доступность

- Прохождение быстрой проверки клавиатурой (Tab) по шапке и боковому меню без ловушек фокуса.
- При необходимости — автоматический прогон `eslint-plugin-jsx-a11y` по `apps/frontend`.

## Тема оформления

- Доля пользователей с нетипичной темой (светлая / тёмная / системная) — опционально, если тема влияет на метрики удержания.

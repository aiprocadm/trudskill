## Итог одной строкой

**Кодового остатка по roadmap почти нет** — все 11 фаз + Pillar A + Wave 1/2 + V1.1 слиты. Критический путь к пилоту лежит через **ops** (восстановить CI, поднять стенд, мониторинг), **юр-оформление** (ЕСИА + НЭП/КЭП) и **бизнес-решения владельца**, а не через написание кода. Инженерный остаток — это активация «спящих» швов (платежи / вебинары / ЭП) под боевые контракты + сверка гос-реестров с эталонами.

> **Обновление 2026-06-23 (§5.141):** все **достижимые из кода** defense-in-depth пункты закрыты — A1 #1 (provider-фильтр), A1 #2 (amount cross-check), A4 (presigned ContentLength + webinar `@Res()` ACK + SCORM actual-bytes budget). Остаток A2/A3/B/C-monitoring/D/E **физически не делается из репозитория** (боевые учётки эквайера, проприетарный КриптоПро SDK, официальные XSD регуляторов, боевой стенд, юр/бизнес-решения) — помечен `ЗАБЛОКИРОВАНО` с исполнителем. CI-блокер **диагностирован** (Actions отключены в настройках репо) — включение за владельцем.

**Минимальный путь к первому пилоту** (без платежей/ЕСИА/вебинаров — они отложены за Phase 0 по решению владельца): **C (CI + стенд + мониторинг + backup) → D1 (инфра) → D3 НЭП (промежуточная модель подписи) → D4 (пилотный контент+клиент) → прогон учебной группы → выдача документов**. Платежи, вебинары, ЕСИА, гос-реестры подключаются после пилота без переработки архитектуры.

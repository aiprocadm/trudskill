# Резервное копирование и откат

## Резерв (рекомендация до первого production)

1. Настроить автоматический **snapshot базы данных PostgreSQL** (pg_dump или снапшоты тома в облаке) не реже одного раза в сутки до появления согласованных RPO.
2. Файловое хранилище объектов / вложений: политику бэкапа согласовать с инфраструктурой (S3 versioning, тома и т.д.).

## Откат релиза

1. Откатить **образ** приложения backend/frontend/worker/realtime к предыдущему digest или тегу.
2. Если миграции ломали схему — использовать заранее подготовленный **обратный SQL** только после проверки на staging; автоматический `down` миграций в проекте может отсутствовать — действовать по решению DBA.

## Связка с задачами плана

Пункт выполняет **TASK-010 / TASK-011** плана запуска: эксплуатационный документ фиксирует минимальные ожидания до получения точных **RPO/RTO** заказчиком (`NFR_LAUNCH_V1.md`).

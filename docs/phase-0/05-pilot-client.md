# WS5 — Пилотный клиент

## Цель

Выбрать 1–2 кандидатов из числа существующих клиентов учебного центра и зафиксировать рамку пилота до начала реального обучения на платформе CDOProf.

---

## Критерии отбора кандидата

> Действующие клиенты у учебного центра уже есть — пилот выбирается из них, а не ищется с нуля.

- Существующий лояльный клиент учебного центра (уже проходил обучение, знает вас лично).
- Небольшая учебная группа — ориентир **10–30 слушателей** (управляемый масштаб для первого запуска).
- Тема курса покрыта **действующей образовательной лицензией** (связь с WS6 — лицензия должна быть оформлена до старта).
- Готовность работать с **промежуточной моделью подписи** (связь с WS3 — например, скачивание PDF вместо полноценной УКЭП на старте).
- Готовность **дать обратную связь** по итогам: заполнить анкету, ответить на звонок или написать короткий отзыв.

## Метрики успеха пилота

По завершении пилота проверьте каждый пункт:

1. **Доля завершивших обучение** — процент слушателей, которые прошли все модули курса (целевое значение согласуйте с клиентом, ориентир ≥ 80 %).
2. **Доля сдавших итоговый экзамен** — процент слушателей, получивших зачёт/оценку «зачтено» (ориентир ≥ 75 %).
3. **Корректность выданных документов** — нумерация по реестру, данные слушателей (ФИО, СНИЛС), QR-код ведёт на правильную запись; нет ни одного документа с ошибкой.
4. **Число обращений в поддержку / инцидентов** — количество технических сбоев или вопросов «не работает», зафиксированных за период пилота (целевое значение — 0 критических).
5. **Отзыв клиента** — NPS-оценка (0–10) или развёрнутый качественный отзыв; зафиксировать письменно.

---

## Чек-лист действий владельца

- [ ] Составить список 2–3 кандидатов из числа текущих клиентов.
- [ ] Выбрать основного кандидата (и запасного) по критериям выше.
- [ ] Согласовать рамку пилота с клиентом: курс, даты, число слушателей, формат документов.
- [ ] Заполнить шаблон рамки пилота (таблица выше) и сохранить в общей папке.
- [ ] Назначить дату старта и внести её в план запуска.

---

## Готово когда

Рамка пилота **заполнена**, клиент **подтвердил участие** (письмо, мессенджер или устно с фиксацией), дата старта **назначена** и внесена в план.

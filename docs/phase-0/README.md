## Критерий «готово»

> Phase 0 завершена, когда первый реальный пилотный клиент (учебный центр / корпоративный заказчик) может провести настоящую учебную группу на CDOProf по публичному адресу, и выданные удостоверения юридически пригодны для применения.

---

## Критический путь

```
Бренд+домен → сервер живой → курс+документы → модель подписи (промежуточная) → прогон группы → выдача документов → (опц.) сдача в реестр
```

## Отложено (вне критического пути)

**ЕСИА-интеграция** и **платежи/54-ФЗ** сознательно вынесены за рамки Phase 0.

- **ЕСИА** — интеграция с Госуслугами уже реализована технически (Phase 4), но для первого пилота не нужна: обучающиеся входят по магической ссылке, идентификация проходит без ЕСИА. Подключение ЕСИА можно добавить позже без переработки архитектуры.
- **Платежи / 54-ФЗ** — онлайн-касса, выдача чеков и соответствующий налоговый учёт отнесены к Phase 7. На пилоте расчёты ведутся по договору и счёту, без приёма оплаты через платформу. Это сознательное решение, а не упущение — оно позволяет сосредоточиться на учебном процессе и не тратить время на сертификацию ПО ККТ до получения первых реальных клиентов.

## Как пользоваться

1. Открывайте документы направлений **в порядке критического пути** (WS1 → WS2 → WS3 → WS4 → WS5 → WS6/7).
2. **WS5 (пилотный клиент)** и **WS6 (лицензия + реестры)** можно начинать **немедленно, параллельно** остальным направлениям — они не зависят от технической готовности сервера.
3. Каждый файл направления содержит конкретный чеклист действий с указанием ответственного и ожидаемого результата.
4. Когда все чеклисты в таблице выше отмечены ☑, Phase 0 считается закрытой.

## Рекомендуемый порядок для владельца

1. **Подтвердить лицензию (WS6)** — первым делом, потому что при провале меняется весь план (см. [06-license-registry-verification.md](06-license-registry-verification.md)).
2. **Параллельно:** выбрать бренд + домен (WS1, [01-brand-domain.md](01-brand-domain.md)) и наметить пилотного клиента (WS5, [05-pilot-client.md](05-pilot-client.md)).
3. **Запустить сервер (WS2)** — [02-server-go-live.md](02-server-go-live.md).
4. **Зафиксировать модель подписи на пилот (WS3)** — [03-signature-model.md](03-signature-model.md).
5. **Настроить пилотный курс (WS4)** — [04-pilot-content.md](04-pilot-content.md).
6. **Получить доступ к нужному реестру (WS7)** — [06-license-registry-verification.md](06-license-registry-verification.md).
7. **Прогон пилота → выдача документов → (опц.) сдача в реестр.**

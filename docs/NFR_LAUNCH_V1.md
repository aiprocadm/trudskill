# NFR первого запуска (v1)

Статус: **зафиксировано как рабочая гипотеза до поступления требований заказчика** (соответствует TASK-002 плана запуска).

## Что задано явно на v1

- **SLA времени доступности**: не задаётся; целевая эксплуатация — «разумная доступность», без финансовых штрафов по SLA.
- **RPO / RTO**: не заданы численно; рекомендуется ежедневный snapshot БД оператором инфраструктуры до появления требований.
- **Пиковая нагрузка / concurrency**: не верифицированы нагрузочными тестами в рамках v1.
- **Резерв**: см. [BACKUP_ROLLBACK.md](./BACKUP_ROLLBACK.md).

## После получения числовых NFR от заказчика

Замените этот файл конкретными значениями (uptime %, минуты простоя в месяц, RPO в часах, RTO в часах, ожидаемое число одновременных пользователей) и добавьте ссылку на результаты нагрузочного прогона.

# NFR первого запуска (v1)

Статус: **MVP-NFR-03 — официальная позиция первого релиза: числовые SLA отсутствуют** (до отдельного согласования с заказчиком). Документ служит снимком для §39 ТЗ («не нарушены SLA» применимо только после появления чисел).

## Что задано явно на v1

- **SLA времени доступности**: не задаётся; целевая эксплуатация — «разумная доступность», без финансовых штрафов по SLA.
- **RPO / RTO**: не заданы численно; рекомендуется ежедневный snapshot БД оператором инфраструктуры до появления требований.
- **Пиковая нагрузка / concurrency**: не верифицированы нагрузочными тестами в рамках v1.
- **Резерв**: см. [BACKUP_ROLLBACK.md](./BACKUP_ROLLBACK.md).

## Формальное утверждение пилота без числовых NFR

Если заказчик принимает первый выкат **без** договорённых uptime %, RPO/RTO и профиля нагрузки, зафиксируйте это в протоколе/приложении к договору и укажите дату здесь: ******\_\_\_******.

## После получения числовых NFR от заказчика

Замените разделы выше конкретными значениями (uptime %, минуты простоя в месяц, RPO в часах, RTO в часах, ожидаемое число одновременных пользователей) и добавьте ссылку на результаты нагрузочного прогона.

# Frontend UX Governance

Дата обновления: 2026-04-14

## 1) Язык интерфейса

- Основной язык UI: русский.
- Допустимы англоязычные термины только для:
  - общеизвестных технических обозначений (`API`, `SLA`, `CSV`, `XLSX`);
  - фирменных имен интеграций и провайдеров.
- Заголовки страниц (`PageHeader.title`), подзаголовки (`subtitle`), CTA-кнопки и статусы должны быть единообразны по языку.

### Базовый глоссарий

- `workspace` -> `Оперативная панель`
- `next actions` -> `Следующие действия`
- `integration settings` -> `Настройки интеграций`
- `sync logs` -> `Журнал синхронизации`
- `inbox tasks` -> `Задачи inbox`
- `blockers` -> `Блокеры`

## 2) Правила PageHeader

- `title`: короткий, предметный, без англицизмов, если не требуется доменный термин.
- `subtitle`: 1-2 предложения про результат для пользователя.
- `actions`: не более 2 первичных действий в header; остальные действия в теле секций.

## 3) Правила таблиц

- Для реестров использовать единый компонент `DataTable`.
- Минимальный контракт для таблицы:
  - сортировка по колонкам, где это возможно;
  - состояние пустого списка;
  - подписи колонок с понятными пользователю терминами;
  - sticky первая колонка для широких таблиц.
- На страницах со значимыми фильтрами состояние фильтров сохранять в URL query (этап внедрения по приоритету модулей).

## 4) Правила фильтров и форм

- Порядок фильтров: текстовый поиск -> статус -> роль/тип -> дополнительные параметры.
- Поля формы должны иметь:
  - `label`;
  - опциональный `hint` для сложных полей;
  - визуализацию ошибки через единый `error` проп.
- Кнопки действий:
  - Primary: сохранить/создать/подтвердить;
  - Secondary: отмена/вспомогательные действия;
  - Danger: разрушительные операции.

## 5) UI-паттерны обязательные для критичных экранов

- Loading / Error / Empty состояния через `state-wrappers`.
- Доступность:
  - все интерактивные элементы доступны с клавиатуры;
  - `aria-label` для неоднозначных кнопок;
  - корректный контраст и `focus-visible`.
- Mobile-ready:
  - фильтры не должны ломать layout на ширине <=1024px;
  - таблицы должны оставаться читаемыми при горизонтальном скролле.

## 6) Приоритет применения

1. `workspace`, `users`, `courses`, `assessment`, `integrations`.
2. `reports`, `notifications`, `documents`.
3. Остальные доменные экраны.

## 7) Definition of Done для UX-изменений

- Нет смешения RU/EN в user-facing копирайте на измененных экранах.
- Для обновленного реестра есть единый набор фильтров и предсказуемые состояния.
- Есть минимум один тест на регрессию поведения (unit/e2e) для измененного critical path.

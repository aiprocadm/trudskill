> **Зачем документ.** Раньше визуальное направление обсуждалось устно и не было нигде записано — между сессиями оно терялось. Этот файл — SSOT по «как выглядит продукт»: цвета, шрифты, ключевые паттерны. Меняется только через новую версию этого файла.

## 1. Проблема

Платформа функционально готова (фазы 1–3, Pillar A, Wave 1–2 слиты), но визуально это «вайрфрейм»:

- **Нет фирменного языка.** Стандартная палитра «синий #2563eb + серый slate» — выглядит «как у всех» / AI-генерация.
- **Шрифт фактически не подключён** — `fontFamily: 'Inter'` в [layout.tsx](../../../apps/frontend/app/layout.tsx) без реальной загрузки → системный fallback.
- **«Следующий шаг» не выделен** — главный элемент пути ученика ([next-step-card.tsx](../../../apps/frontend/src/features/learner-home/next-step-card.tsx)) визуально неотличим от соседних карточек. Это прямо бьёт по боли №1 «неудобно ученикам».

## 2. Концепция: надёжная официальность, читаемость в приоритете

Продукт существует ради выдачи **государственных удостоверений** — отсюда ощущение «надёжно и официально, как госуслуга». Носитель доверия — **глубокий тёмно-синий** (герой, навигация) на **чистом холодном фоне**; первичные действия — **читаемый синий**, кнопка героя — **белая на синем** (максимальный контраст).

Это сознательно **сдержанная/институциональная** эстетика (не игровая edtech, не брутализм). Отличительность — за счёт точности исполнения, типографики и тёмно-синего героя, а не яркого цвета.

> **Ревизия 2026-06-06 (по фидбэку владельца).** Первый вариант использовал «золото печати» как акцент — владелец отметил **плохую читаемость**. Золото убрано полностью: на светлом фоне его контраст ~3:1 (ниже порога). Палитра пересобрана под читаемость, контраст всех пар **проверен числами** (см. §7): тело текста 18:1 (светлая) / 14.6:1 (тёмная), минимум по палитре — 5.37:1 (бейдж), всё ≥ WCAG AA.

## 3. Опорные решения

### 3.1 Типографика

- **Golos Text** (Paratype) — весь функциональный UI и текст. Отличный кириллический шрифт с «госуслуговой» ДНК; крупный, читаемый (среди учеников много возрастных/производственников).
- **PT Serif** (Paratype) — вордмарк и заголовок-герой «Следующего шага». Даёт «дипломную» весомость. Применяется **только акцентно**, не в функциональном UI.
- Загрузка через `next/font/google` → шрифты **self-hosted** в нашем бандле (end-user не обращается к Google → довод по 152-ФЗ).
- Масштаб: крупнее и плотнее заголовки, комфортный line-height в тексте.

### 3.2 Палитра (через CSS-переменные, имена существующих токенов сохранены)

- **Navy (brand):** глубокий тёмно-синий — герой и навигация; **читаемый синий** (`#1e40af`) — первичные действия, ссылки, активные состояния, прогресс.
- **Нейтрали:** холодные чистые (cool gray) — фон чуть серее белого, чтобы карточки «выходили вперёд». Текст — почти чёрный (`#0f1626`).
- **Кнопка героя — белая** на тёмно-синем (контраст 14:1). Акцент-цвет-«украшение» убран ради читаемости.
- **Боковая навигация — тёмно-синяя «рейка»** даже в светлой теме (рамка вокруг каждого экрана, **белый** активный пункт) — узнаваемая фирменная подпись.
- Светлая и тёмная темы; семантические статусы (success/warning/danger) — углублены до AA-контраста.

### 3.3 Ключевой паттерн — Hero «Следующий шаг»

Из обычной карточки → в **герой-блок**: тёмно-синий градиент, светло-голубая «надпечатка» (eyebrow «СЛЕДУЮЩИЙ ШАГ»), крупный serif-заголовок действия, заметная **белая** кнопка, тонкая декоративная «печать» (гильош) на фоне. Однозначно считывается как «вот что делать сейчас».

### 3.4 Поверхности и движение

- Карточки/отступы/тени — выверенный ритм, чуть больше радиус, мягкие слоистые тени.
- Прогресс-бары курсов стилизуются (сейчас — дефолтный браузерный `<progress>`).
- Микро-движение: сдержанное появление карточек на главной (stagger), мягкий hover. Уважать `prefers-reduced-motion`.

### 3.5 Доступность (Phase 10, делаем сразу)

- Контраст AA+; `focus-visible` усилен; крупный кегль; навигация с клавиатуры (уже есть в shell — сохранить).

## 4. Рычаг реализации

Всё выражается через **существующую систему токенов** — [tokens/index.ts](../../../packages/ui/src/tokens/index.ts) (значения переменных) + [foundation.ts](../../../packages/ui/src/styles/foundation.ts) (базовый CSS). Поскольку весь UI (сайдбар, топбар, кнопки, карточки) читает эти переменные, новый стиль **расходится по всем ~40 экранам автоматически**. Имена токенов не меняем — меняем значения и добавляем несколько новых (золото, hero).

## 5. План раскатки

1. **Срез-валидация (этот заход):** шрифты + токены + foundation + герой → **главный экран ученика** (`/learner`). Скриншот светлая/тёмная для подтверждения направления.
2. **После подтверждения:** отдельный план (`writing-plans`) на сверку остальных кабинетов (админ, преподаватель), стилизацию таблиц/форм/модалок под новый язык, аудит контраста (axe), визуальные снапшоты эталонных экранов (см. [UI_SUCCESS_METRICS.md](../../../apps/frontend/docs/UI_SUCCESS_METRICS.md)).

## 6. Явно вне scope (пока)

- Полный ребрендинг (логотип, домен, название) — бизнес-решение владельца.
- Иллюстрации/иконочный сет на заказ — отдельно.
- Замена компонентной архитектуры (CSS-in-JS токены остаются) — не трогаем.
- Переписывание контента/копирайта — правила [FRONTEND_UX_GOVERNANCE.md](../../FRONTEND_UX_GOVERNANCE.md) в силе.

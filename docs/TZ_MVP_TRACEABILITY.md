## MVP-SCOPE-02: официальная граница MVP (пилот)

До письменного решения заказчика действует **рабочая презумпция** из [SDOPROF_TZ_FINAL.md](../SDOPROF_TZ_FINAL.md) §873–889: в пилот входит рекомендуемый MVP; ниже — области, требующие явного «вкл/выкл» в контракте.

| Область                                                           | Рекомендация для пилота            | Решение заказчика (дата)         |
| ----------------------------------------------------------------- | ---------------------------------- | -------------------------------- |
| SSO / LDAP (BL-009, этап 9)                                       | вне пилота без уточнения протокола | _заполнить: в scope / вне scope_ |
| Заявки и документооборот (этап 10)                                | вне пилота                         | _заполнить_                      |
| Юридически значимые документы / ЭП / legal log (§867–871, BL-011) | вне пилота без юр. уточнения       | _заполнить_                      |
| Компетенции, ИПР, траектории (этап 7)                             | post-MVP                           | _заполнить_                      |
| AI (этап 12, BL-012)                                              | post-MVP                           | _заполнить_                      |

После заполнения столбца «Решение заказчика» обновите строки в таблице BL выше при необходимости.

## Scope MVP — требует подтверждения заказчиком

- SSO/LDAP, юридически значимые документы, заявки/согласования, точные интеграции — помечены в SDOPROF как «Требует уточнения»; до ответа заказчика считаются вне минимального пилота, если явно не согласовано.

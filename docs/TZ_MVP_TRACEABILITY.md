## MVP-SCOPE-02: официальная граница MVP (пилот)

Источник решений: **[SDOPROF §47 (приложение Б)](../SDOPROF_TZ_FINAL.md#47-приложение-б-фиксация-границ-пилота-must--вне-scope)**, дата фиксации в репозитории **2026-05-03**. До подписи заказчиком это **рабочая презумпция исполнителя**; см. также [§2.5 ТЗ](../SDOPROF_TZ_FINAL.md#25-рекомендуемый-состав-mvp-пилот).

### Must (BL-001, BL-002, BL-003, BL-004, BL-005, BL-006, BL-007, BL-010)

**Решение для пилота: все Вкл.** Детальные комментарии — в приложении Б §47.2; дублировать полную таблицу здесь не требуется.

### Темы вне пилота / post-MVP и опциональная зона

| Область                                                 | Рекомендация ТЗ (до решения)            | Решение зафиксировано (рабочая презумпция, 2026-05-03)                   |
| ------------------------------------------------------- | --------------------------------------- | ------------------------------------------------------------------------ |

## Как поддерживать этот файл

0. Если неясно, **куда писать изменение** — сначала [DOCUMENTATION_MAP.md](./DOCUMENTATION_MAP.md).
1. При изменении API под BL-003 / BL-007 / BL-008 — обновить таблицу и [Handoff §21](../LMS_AGENT_HANDOFF.md#21-новые-mvp-api-быстрый-справочник).
2. После встречи с заказчиком: перенести решения из заполненного [приложения Б SDOPROF (§47)](../SDOPROF_TZ_FINAL.md#47-приложение-б-фиксация-границ-пилота-must--вне-scope) в колонку **MVP-SCOPE-02** выше и при необходимости обновить [SDOPROF §41](../SDOPROF_TZ_FINAL.md#41-backlog-укрупнённый).
3. Термины в тексте трассировки согласовать с [глоссарием (приложение А)](../SDOPROF_TZ_FINAL.md#46-приложение-а-глоссарий-терминов-тз).
4. Не дублировать длинные инструкции запуска: отсылка к [README](../README.md) и [LAUNCH_RUNBOOK](./LAUNCH_RUNBOOK.md).

## Шаги для запуска smoke (для оператора)

Минимальный сценарий-стенд:

1. Создать тестовый тенант (или использовать существующий `tenant_demo`).
2. Создать тестового ученика + enrollment + group_course.
3. Поднять local dev:
   ```bash
   pnpm -w dev
   ```
4. По каждому сценарию runbook'а — выполнить шаги «Симптом → Проверки → Действия → Verify» руками через API (curl/Postman) или admin UI.
5. Заполнить чек-лист выше реальными результатами.
6. Сделать коммит `docs(runbook): smoke results for ...`.

Для сценария §5 (компрометация) — **не запускать на prod**. Только на dev/staging.
